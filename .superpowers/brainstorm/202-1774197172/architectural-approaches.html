Frontend Architecture Approaches

How to structure the dashboard and embeddable widget

A

Monolithic Next.js App

Single Next.js application with widget as an iframe-able route

📁 Project Structure
frontend/
├── app/
├── (dashboard)/
├── chatbots/
├── [id]/
├── documents/
├── design/
├── analytics/
├── (widget)/
├── widget/[botId]/
├── components/
├── dashboard/
├── widget/

Pros

  • Single codebase, easier maintenance
  • Shared components between dashboard & widget
  • Simpler deployment (one app)

Cons

  • Widget inherits Next.js bundle size
  • Iframe has some limitations
  • Less isolation between widget and dashboard
B

Separate Widget Bundle

Next.js dashboard + Standalone widget built with Vite (vanilla JS/React)

📁 Project Structure
frontend/
├── dashboard/ (Next.js 14)
├── app/
├── components/
├── widget/ (Vite + React)
├── src/
├── ChatWidget.tsx
├── ChatBubble.tsx
├── dist/
├── chatbot-widget.js (~50KB)
├── shared/
├── types/
├── api-client/

Pros

  • Widget is lightweight (~50KB gzipped)
  • Widget loads independently, no Next.js overhead
  • Better performance on customer websites
  • Clear separation of concerns

Cons

  • Two codebases to maintain
  • Need to sync shared types/utils
  • More complex build pipeline
C

Hybrid with Shared Design System

Next.js dashboard + Standalone widget + Shared UI package

📁 Project Structure
frontend/
├── packages/
├── ui/ (Shared components)
├── glass-card/
├── gradient-button/
├── chat-bubble/
├── types/ (Shared types)
├── api-client/ (Shared API)
├── apps/
├── dashboard/ (Next.js)
├── widget/ (Vite bundle)
├── package.json (workspace)

Pros

  • Consistent design system across all apps
  • Maximum code reuse
  • Lightweight widget bundle
  • Scalable for future apps (mobile, etc.)

Cons

  • Higher initial setup complexity
  • Monorepo tooling (Turborepo/pnpm workspaces)
  • More upfront investment
💡 My Recommendation: Option B (Separate Widget Bundle)

For a SaaS chatbot platform where the widget is embedded on customer websites, a separate lightweight bundle is essential. Next.js is great for the dashboard, but the widget should be a standalone ~50KB bundle that loads fast on any website. This gives you the best balance of performance and maintainability without the complexity of a full monorepo setup.
